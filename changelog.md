## 1.0.0 - 2026-04-07
* The `CallControllerFindAllPaginatedRequest` class and the `CallControllerFindAllPaginatedRequestSortOrder` enum have been removed from the SDK. Consumers using these types to list calls should migrate to the updated paginated call listing API — please refer to the latest documentation for the replacement request type.
* The following public types have been removed from the SDK: `GenerateStructuredOutputSuggestionsDto`, `CreateVoicemailToolDtoType`, and `UpdateSupabaseCredentialDto`. Additionally, `AnalyticsClient.GetAsync` now returns `WithRawResponseTask<IEnumerable<AnalyticsQueryResult>>` instead of `Task<IEnumerable<AnalyticsQueryResult>>`; callers must update their awaiting code and can now access the raw HTTP response via `.Result.RawResponse`.
* All `AssistantsClient` async methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Callers must update their code to await the new return type, e.g. `var result = await client.Assistants.ListAsync(...); var data = result.Data;`. The new return type also exposes `.RawResponse` for access to the HTTP status code, response headers, and request URL. Additionally, a new `AssistantSpeechStarted` value has been added to the `UpdateAssistantDtoClientMessagesItem` enum.
* The `CallControllerFindAllPaginatedAsync` method has been removed from `CallsClient`. All `CallsClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`, giving access to the raw HTTP response (status code, URL, headers) alongside the deserialized result — callers must update usages accordingly (e.g. `await client.Calls.ListAsync(request)` now returns a `WithRawResponseTask<IEnumerable<Call>>`; use `.Result` or `await` then `.Data` to access the payload). The `AssistantSpeechStarted` value has been added to the `UpdateAssistantDtoServerMessagesItem` enum.
* All `CampaignsClient` async methods (e.g. `CampaignControllerFindAllAsync`, `CampaignControllerCreateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`, giving consumers access to HTTP status codes, headers, and the request URL via `.RawResponse`. Additionally, `CreateCampaignDto.PhoneNumberId` is no longer `required` (now `string?`) and `Customers` is now nullable — update any object initializers that relied on these being required/non-null. New optional `SquadId` and `DialPlan` properties are available on both `CreateCampaignDto` and `UpdateCampaignDto`.
* All `ChatsClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `CreateResponseAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Awaiting the result yields a `WithRawResponse<T>` object with a `.Data` property containing the deserialized response and a `.RawResponse` property exposing the HTTP status code, URL, and headers. Existing code that awaits these methods directly will continue to work; code that assigns the return value to a `Task<T>` variable must be updated to use `WithRawResponseTask<T>`. Additionally, `ListChatsRequest` gains two new optional filter fields: `Id` and `AssistantIdAny`.
* The SDK now supports `Retry-After` and `X-RateLimit-Reset` response headers to intelligently determine retry delays, with added jitter for better distribution. `VapiClientApiException` now accepts an optional `innerException` parameter for improved error context. JSON serialization has been enhanced with relaxed-escaping overloads and improved handling of `Optional`, `Nullable`, and extension data fields.
* The `EvalClient` methods now return `WithRawResponseTask<T>` instead of `Task<T>`, providing access to the raw HTTP response (status code, URL, and headers) via the `.RawResponse` property on the awaited result. Callers that previously awaited these methods directly can continue to do so — the awaited result now exposes both `.Data` (the deserialized response) and `.RawResponse` (the raw HTTP metadata). Deserialization errors now throw `VapiClientApiException` (including status code and response body) instead of the more generic `VapiClientException`.
* The `FilesClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`, providing access to the underlying HTTP response alongside the deserialized result. Callers that previously awaited these methods directly (e.g., `await client.Files.ListAsync()`) must now access the `.Data` property of the result or use `.Result` on the `WithRawResponseTask<T>` wrapper. Additionally, deserialization errors from `FilesClient` now throw `VapiClientApiException` instead of `VapiClientException`.
* All `InsightClient` methods now return `WithRawResponseTask<T>` instead of `Task<T>`, providing access to the raw HTTP response (status code, headers, and URL) via the `.RawResponse` property, while the deserialized result remains available via `.Data`. Callers must update `await client.Insight.InsightControllerFindAllAsync(...)` and similar calls to use the new return type — for example, `var result = await client.Insight.InsightControllerFindAllAsync(...); var data = result.Data;`. Additionally, deserialization failures now throw `VapiClientApiException` instead of `VapiClientException`.
* The `ObservabilityScorecardClient` methods (`ScorecardControllerGetAsync`, `ScorecardControllerRemoveAsync`, `ScorecardControllerUpdateAsync`, `ScorecardControllerGetPaginatedAsync`, and `ScorecardControllerCreateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Callers can use `.Result` or `await` as before to get the typed data, or access `.RawResponse` for the HTTP status code, URL, and headers of the underlying response.
* All methods on `PhoneNumbersClient` (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`, `PhoneNumberControllerFindAllPaginatedAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Callers must update their code to await the `.Result` or `.Data` property of the returned value to access the deserialized response. The raw HTTP response (status code, headers, URL) is now accessible via `.RawResponse`.
* All `ProviderResourcesClient` methods now return `WithRawResponseTask<T>` instead of `Task<T>`, giving callers access to the HTTP status code, headers, and raw response alongside the deserialized result. **Existing code that awaits these methods directly will need to be updated** — use `.Result` or `await` the returned `WithRawResponseTask<T>` and access `.Data` for the deserialized value.
* Additionally, the `Cartesia` provider is now supported: a new `Cartesia` enum value has been added to the provider enums used in Create and Delete operations.
* The SDK now supports `Cartesia` as a provider option across all provider resource request types (`Get`, `GetPaginated`, and `Update`). `CreateSessionDto` gains two new optional properties — `AssistantOverrides` (for variable substitution and configuration overrides) and `CustomerId` (to associate a customer by ID). `ListSessionsRequest` gains five new optional filter fields: `Id`, `AssistantIdAny`, `CustomerNumberAny`, `PhoneNumberId`, and `PhoneNumberIdAny`, enabling more precise session queries.
* The `SessionsClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. This gives callers access to raw HTTP response metadata (status code, URL, and headers) via the `.RawResponse` property, while the deserialized result remains accessible via `.Data`. Existing code that `await`s these methods directly will continue to work unchanged (since `WithRawResponseTask<T>` is implicitly awaitable and returns `T`), but any code that stores or passes the return value as a `Task<T>` must be updated to use `WithRawResponseTask<T>`.
* Additionally, `ListAsync` now supports five new optional query filters: `Id`, `AssistantIdAny`, `CustomerNumberAny`, `PhoneNumberId`, and `PhoneNumberIdAny`.
* All `SquadsClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Callers must update their code to await the result and access `.Data` for the deserialized response, or use `.WithRawResponse` to retrieve the full HTTP response metadata. Additionally, `UpdateStructuredOutputDto` gains two new optional properties — `Type` and `Regex` — enabling regex-based structured output extraction.
* All `StructuredOutputsClient` methods now return `WithRawResponseTask<T>` instead of `Task<T>`, providing access to the raw HTTP response (status code, headers, and request URL) via `.RawResponse` alongside the deserialized data via `.Data`. Existing callers that awaited these methods directly must be updated to await `.Data` or use the implicit conversion. The `StructuredOutputControllerSuggestAsync` method has been removed with no direct replacement.
* **Breaking changes in this release:**
* All `ToolsClient` methods (`ListAsync`, `CreateAsync`, `GetAsync`, `DeleteAsync`, `UpdateAsync`) now return `WithRawResponseTask<T>` instead of `Task<T>`. Call `.Result` or `await` the inner task to retrieve the deserialized data via the `.Data` property, or access `.RawResponse` for HTTP metadata. Update all call sites accordingly.
* The `AdditionalProperties` property on response record types (e.g. `AnalysisCost`, `Analysis`, `AnalysisPlan`, etc.) has changed from `IDictionary<string, JsonElement>` to `ReadOnlyAdditionalProperties`. Code that wrote to or cast this property must be updated.
